# USER_INTERVIEWS.md

## Interview 1

### Participant
Ujjwal — Full Stack Developer working with early-stage SaaS startups

### Discussion Summary
The participant regularly uses ChatGPT Plus, Cursor, and GitHub Copilot for development workflows. He mentioned that many small teams subscribe to multiple AI tools without evaluating whether they actually need all of them.

### Key Quotes
- “Most startups honestly don’t track AI tool spending seriously until it becomes a noticeable monthly cost.”
- “Your UI already feels very clean and modern compared to most student projects.”
- “The savings section is the strongest part because it immediately shows value.”
- “I’d trust the recommendations more if there were clearer explanations for why the system suggests switching tools.”

### Most Surprising Insight
The participant said overlapping subscriptions are often caused by teams experimenting with multiple AI tools at the same time rather than intentionally overspending.

### Design Feedback
Positive feedback:
- Clean SaaS-style UI
- Good dashboard structure
- Easy-to-understand audit flow

Suggested improvements:
- Add more tool logos and visual branding
- Improve recommendation explanations
- Add stronger “before vs after” comparison cards

### Changes Made After Interview
- Improved recommendation reasoning in the audit engine
- Added more detailed savings explanations
- Planned improvements for visual branding and recommendation cards

## Interview 2

- Add more animations and hover effects
- Make the recommendation cards visually more prominent

### Changes Made After Interview

## Interview 3

### Participant
Shivraj — Small Startup Founder managing a 6-person development team

### Discussion Summary
The participant currently pays for multiple AI tools including Claude, ChatGPT Team, and GitHub Copilot. He mentioned that AI spending grows very quickly once multiple employees begin using different subscriptions independently.

### Key Quotes
- “I actually don’t know whether our current AI spending is reasonable or not.”
- “The dashboard gives a good overview quickly without overwhelming the user.”
- “I would definitely use this if the recommendations were financially trustworthy.”
- “The UI already looks polished enough to be shared publicly.”

### Most Surprising Insight
The participant said trust and financial reasoning are more important than AI-generated recommendations themselves.

### Design Feedback
Positive feedback:
- Professional SaaS appearance
- Strong dashboard layout
- Good visual hierarchy

Suggested improvements:
- Add more financial reasoning behind recommendations
- Include benchmark comparisons
- Add downloadable reports in the future

### Changes Made After Interview
- Improved recommendation explanations with more financial reasoning
- Refined savings calculation logic
- Planned future benchmark comparison features
